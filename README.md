# My main goal in this project is to design an airport-like system. For this reason, I designed a system where you can add flights to the system, list all flights according to departure and departure location, and buy tickets. If I need to explain the operation and use of the system a little: first of all, "username" and "password" that I defined in the code must be entered in order to be able to login to the system. If the user does not enter the password and user name correctly, a warning message appears and is sent to the password entry screen again. After entering the password, a menu welcomes the user. This menu consists of 4 sections. These sections are 1-Add Flight, 2-Search Flight, 3-Buy Ticke, 4-Exit. 1- In this section, the administrator can add flights to the system. It takes information such as flight number, departure point, landing point, departure time, estimated landing time, number of seats and company name from the user and prints it in a file. 2- By entering the departure and landing points, the user can see all the details of all flights between those points. 3- My aim at this point is to create a ticket sales simulation. You are asked how many seats you want to buy from the flight whose flight number is to be entered. If the number of tickets requested is less than or equal to the number of empty seats, the sale takes place. 3 payment methods are offered to the user for sales. I used the code I used in my previous projects by changing it here again. The payment methods offered to the user are respectively credit card, paypal and cash payment. If the user chooses credit card or paylap, the system will not allow payment unless he enters a 16-digit card number and a 4-digit Password. With 4-option, the user can exit the system from the menu. It is redirected to the password entry screen again. Thank you in advance for your interest and likes.




# Mein Hauptziel in diesem Projekt ist es, ein flughafenähnliches System zu entwerfen. Aus diesem Grund habe ich ein System entwickelt, in dem Sie Flüge zum System hinzufügen, alle Flüge nach Abflug und Abflugort auflisten und Tickets kaufen können. Wenn ich die Bedienung und Nutzung des Systems ein wenig erläutern muss: Zunächst müssen „Benutzername“ und „Passwort“ eingegeben werden, die ich im Code definiert habe, um mich am System anmelden zu können. Wenn der Benutzer das Passwort und den Benutzernamen nicht richtig eingibt, erscheint eine Warnmeldung und wird erneut zum Bildschirm für die Passworteingabe gesendet. Nach Eingabe des Passworts begrüßt ein Menü den Benutzer. Dieses Menü besteht aus 4 Abschnitten. Diese Abschnitte sind 1-Flug hinzufügen, 2-Flug suchen, 3-Tick kaufen, 4-Aussteigen. 1- In diesem Abschnitt kann der Administrator dem System Flüge hinzufügen. Es nimmt Informationen wie Flugnummer, Abflugort, Landepunkt, Abflugzeit, voraussichtliche Landezeit, Anzahl der Sitzplätze und Firmenname vom Benutzer und druckt sie in eine Datei. 2- Durch Eingabe der Abflug- und Landepunkte kann der Benutzer alle Details aller Flüge zwischen diesen Punkten sehen. 3- Mein Ziel an dieser Stelle ist es, eine Ticketverkaufssimulation zu erstellen. Sie werden gefragt, wie viele Plätze Sie von dem Flug kaufen möchten, dessen Flugnummer eingegeben werden soll. Wenn die Anzahl der angeforderten Tickets kleiner oder gleich der Anzahl der leeren Plätze ist, findet der Verkauf statt. 3 Zahlungsmethoden werden dem Benutzer für den Verkauf angeboten. Ich habe den Code verwendet, den ich in meinen vorherigen Projekten verwendet habe, indem ich ihn hier erneut geändert habe. Als Zahlungsarten werden dem Nutzer jeweils Kreditkarte, Paypal und Barzahlung angeboten. Wenn der Benutzer Kreditkarte oder Paylap wählt, erlaubt das System keine Zahlung, es sei denn, er gibt eine 16-stellige Kartennummer und ein 4-stelliges Passwort ein. Mit 4-Option kann der Benutzer das System über das Menü verlassen. Es wird wieder auf die Passwort-Eingabemaske umgeleitet. Vielen Dank im Voraus für Ihr Interesse und Ihre Likes.
